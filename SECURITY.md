# Security Policy

PackLock encrypts user data, so security issues are taken seriously.

## Reporting a Vulnerability

If you discover a security vulnerability (e.g. a flaw in the encryption
scheme, key derivation, or archive handling), please **do not open a public
GitHub issue**. Instead, report it privately by emailing:

**engr.abdullah.hamza@gmail.com**

Please include:
- A description of the vulnerability and its potential impact.
- Steps to reproduce, or a proof-of-concept if available.
- The version/commit of PackLock affected.

You should receive an acknowledgment within a few days. Once a fix is
available, we'll coordinate on disclosure timing and credit you in the release
notes, if you'd like.

## Supported Versions

PackLock is currently pre-1.0 and does not yet maintain separate release
branches. Please report issues against the latest commit on `main`.
